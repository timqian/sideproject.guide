# How to realize your idea quickly

When you have ignited enough enthusiasm for your idea, and can't help pressing the keyboard. Before starting to change the world with code, calm down first.

## You don't have to code

Each line of code needs to be maintained, everytime when you add a line, the cost of your side project gets higher. It will be perfect if it is possible to solve the problem without code.

Correct priority of solve problems:

1. No code.
2. Write only frontend code.
3. If a backend is required, use Serverless 
programme in a way that can be built serverlessly method.
4. Maintain a server.

In short, for the development, the goal of the optimization is to make the work less for now and future. If you have to use code to solve the problem, select your most familiar tools.

## 第一版做到什么程度

- 是可用的，帮助用户解决了问题，或者提供了价值
- 提供渠道供用户反馈
- 花费时间最好不要超过一个月(因人的耐心而异)，以便及时获得用户反馈

## 我使用的一些建筑工具

- [TailwindCSS](https://tailwindcss.com/): 使用 class 来定义样式，比直接写 CSS 方便一些
- [React](https://reactjs.org/): 模块化构建前端项目
- [Nextjs](https://nextjs.org/): 用 React 写多页应用更方便，性能优秀
- [Serverless](https://serverless.com/): 免运维
- [Vercel](https://vercel.com/): 一键部署 Nextjs 等应用，性能有保障
- [Dynamodb](https://aws.amazon.com/cn/dynamodb): Serverless 数据库
